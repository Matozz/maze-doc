# Tooltip

When activated, Tooltips display a text label identifying an element, such as a description of its function.

## Basic tooltip

<Wrapper>
  <Tooltip title="Show the full content">
    <Button variant="text">Show</Button>
  </Tooltip>
</Wrapper>

```jsx
<Tooltip title="Show the full content">
  <Button variant="text">Show</Button>
</Tooltip>
```

## Positioned tooltips

The Tooltip has 4 placements choice (will support 12 placements in future releases). They don't have directional arrows; instead, they rely on motion emanating from the source to convey direction.

<Wrapper>
  <Grid alignItems="center" direction="column" spacing={4}>

      <Button variant="text">Top</Button>
    </Tooltip>
    <Grid spacing={6}>
      <Tooltip title="Left" position="left">
        <Button variant="text">Left</Button>
      </Tooltip>
      <Tooltip title="Right" position="right">

      </Tooltip>
    </Grid>

      <Button variant="text">Bottom</Button>
    </Tooltip>

</Wrapper>

## Arrow tooltips

You can use the `arrow` prop to give your tooltip an arrow indicating which element it refers to.

<Wrapper>
  <Grid alignItems="center" spacing={2}>

      <Button variant="text">Top</Button>
    </Tooltip>
    <Tooltip title="Bottom" position="bottom" arrow>
      <Button variant="text">Bottom</Button>
    </Tooltip>
  </Grid>
</Wrapper>

```jsx
<Grid alignItems="center" spacing={2}>
```

## Customization

Here are the example of customizing the component. You can customize the tooltip content by passing elements to the `title` prop.

<Wrapper>
  <Tooltip
    title={
      <div>
        Tooltip with HTML
        <br />
        <em>{"And here's"}</em> <b>{"some"}</b> <u>{"amazing content"}</u>.{" "}
        {"It's very engaging. Right?"}
      </div>
    }
    containerStyle={{
      background: "#f5f5f5",
      color: "black",
      border: "#dbdbdb 1px solid",
      padding: 10,
      fontSize: 14,
      width: 240,
    }}
    arrowStyle={{
      color: "#dbdbdb",
    }}
    position="right"
    arrow
  >
    <Button variant="text">HTML</Button>
  </Tooltip>
</Wrapper>

```jsx
<Tooltip
  title={
    <div>
      Tooltip with HTML
```

## Disabled elements

By default disabled elements like `<button>` do not trigger user interactions so a `Tooltip` will not activate on normal events like hover. To accommodate disabled elements, add a simple wrapper element, such as a `span` and disabled `pointerEvents` on `<button>`.

<Wrapper>
  <Tooltip
    title="You don't have the permission to activate"
    position="bottom"
    arrow
  >
    <span>
      <Button variant="text" disabled style={{ pointerEvents: "none" }}>
        Activate
      </Button>
    </span>
  </Tooltip>
</Wrapper>

:::info Note

By default, we add the `not-allowed` cursor style to the `Input` element, such as `<Button>`, `<Radio>` and `<CheckBox>`, which remain the disabled button to have pointer-events. So simply add set there pointerEvents to `"none"` can solve this issue.

:::

```jsx
<Tooltip
  title="You don't have the permission to activate"
  position="bottom"
  arrow
>
  <span>
    <Button variant="text" disabled style={{ pointerEvents: "none" }}>
      Activate
    </Button>
  </span>
</Tooltip>
```
